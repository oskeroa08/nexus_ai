## 1. Введение

В современном мире голосовые ассистенты стали неотъемлемой частью повседневной жизни. Они упрощают взаимодействие с устройствами, позволяют быстро получать информацию и выполнять команды hands-free. Цель данного проекта – разработка локального голосового ассистента для Windows с современным графическим интерфейсом, поддержкой офлайн синтеза речи и возможностью гибкой настройки пользовательских команд.

Nexus AI сочетает в себе следующие ключевые особенности:
- Распознавание речи с wake word
- Офлайн синтез речи (Silero) и онлайн (Edge TTS)
- Интеграция с ИИ-провайдерами (OpenAI, Groq)
- Пользовательские команды и профили
- Современный интерфейс с визуализацией

---

## 2. Общая характеристика проекта

Проект Nexus AI – это десктопное приложение для Windows, разработанное на языке Python с использованием фреймворка PySide6 для создания графического интерфейса.

Основные задачи, решаемые приложением:
1. Непрерывное прослушивание аудио с микрофона и детекция wake word
2. Распознавание речевых команд
3. Синтез ответов
4. Выполнение системных команд (открытие приложений, сайтов, управление громкостью, закрытие процессов)
5. Генерация интеллектуальных ответов с помощью ИИ
6. Управление пользовательскими командами и профилями

---

## 3. Анализ предметной области

На рынке существуют множество голосовых ассистентов (Cortana, Alice, Siri, Google Assistant), но большинство из них являются облачными и не предоставляют возможности гибкой настройки или работы полностью офлайн. Также часто отсутствует поддержка создания пользовательских команд и профилей.

Проект Nexus AI направлен на решение этих недостатков:
- Поддержка офлайн ТТС (Silero)
- Возможность полностью локальной работы (без облачных сервисов)
- Гибкий редактор пользовательских команд
- Управление профилями команд
- Современный и привлекательный интерфейс

## 5. Выбор средств разработки

Для разработки были выбраны следующие инструменты и библиотеки:
- **Язык программирования:** Python 3.13
- **GUI фреймворк:** PySide6 (кроссплатформенный, современный)
- **Распознавание речи:** SpeechRecognition (обертка для Google STT)
- **Синтез речи:**
  - Silero (офлайн, ru-RU)
  - Edge TTS (онлайн, много голосов)
  - pyttsx3 (офлайн fallback)
- **ИИ-провайдеры:** OpenAI API, Groq API
- **Системные команды:** pynput (клавиши), psutil (процессы), webbrowser (сайты)
- **Сборка EXE:** PyInstaller
- **Архиватор зависимостей:** pip (requirements.txt)

## 6. Проектирование программного обеспечения

При проектировании был выбран модульный подход с четким разделением ответственности:
- **core:** логика приложения (распознавание, синтез, команды)
- **gui:** графический интерфейс
- **assets:** внешние ресурсы (Silero модель, иконки)

---

## 7. Архитектура Nexus AI

Архитектура приложения многослойная:
1. **GUI слой (main_window.py, commands_dialog.py, settings_window.py):** Отображает интерфейс, взаимодействует с пользователем
2. **Слой логики (core/):**
   - audio_manager.py – управление конфигом и аудио
   - speech_engine.py – распознавание и синтез речи
   - system_commands.py – выполнение команд
   - command_manager.py – управление пользовательскими командами и профилями
   - ai_handler.py – интеграция с ИИ-провайдерами
3. **Внешние сервисы:** Silero, Google STT, Edge TTS, OpenAI/Groq

## 8. Описание всех модулей

### Модуль core/

#### 1. audio_manager.py
Управляет конфигурацией приложения (загрузка/сохранение config.json), запускает/останавливает мониторинг уровня аудио.

#### 2. speech_engine.py
Основной модуль для работы с речью:
- **listen:** Слушает микрофон, возвращает распознанный текст
- **extract_command:** Извлекает команду из текста после wake word
- **speak:** Синтезирует речь (выбирает провайдера, запускает в отдельном потоке)
- **preload_tts:** Предзагружает Silero в фоне
- **_play_file:** Проигрывает аудио с помощью PowerShell (скрыто, без консоли)

#### 3. system_commands.py
Обрабатывает команды: сначала пытается найти пользовательскую, потом проверяет стандартные (время, дата, приветствие), потом передает запрос ИИ.

#### 4. command_manager.py
Менеджер пользовательских команд и профилей:
- Хранение в JSON файлах
- Добавление/изменение/удаление команд
- Управление профилями
- Выполнение команд по тексту

#### 5. ai_handler.py
Интеграция с OpenAI и Groq для генерации ответов.

### Модуль gui/

#### 1. main_window.py
Главное окно приложения:
- **Voice Worker:** Фоновый поток для прослушивания микрофона
- **TTS Worker:** Фоновый поток для синтеза речи
- **Chat History:** История диалога с сообщениями и ошибками
- **Top Bar:** Заголовок, статус, кнопки управления
- **Visualizer:** Анимированная сфера, реагирующая на звук

#### 2. commands_dialog.py
Диалог для управления пользовательскими командами и профилями:
- Список команд
- Редактор команд (триггер, действие, цель, ответ)
- Управление профилями

#### 3. chat_history.py
Виджет истории чата:
- ChatBubble (сообщения пользователя и ассистента)
- ErrorMessage (отображение ошибок в чате)

#### 4. visualizer.py
Визуализатор аудио в виде анимированной сферы с кольцами и частицами.

## 9. Описание интерфейса

Интерфейс выполнен в стиле glassmorphism (прозрачные панели, размытие, неоновые акценты):
1. **Верхняя панель:**
   - Заголовок "NEXUS AI"
   - Статусная метка ("Слушаю...", "Думаю...")
   - Кнопки: пауза микрофона, отключить звук, команды, настройки
2. **Визуализатор:** Анимированная сфера в центре
3. **История диалога:** Прокручиваемая область с сообщениями (пользователь справа, ассистент слева, ошибки по центру)

Цветовая палитра: Темный фон (0x05070D), акценты голубого цвета (0x57E8FF).

## 10. Руководство пользователя

### Установка и запуск
1. Установите зависимости: `pip install -r requirements.txt`
2. Скачайте Silero модель: `python download_silero.py`
3. Запустите: `python main.py`

### Основные функции
1. **Активация:** Скажите "Нексус", затем команду
2. **Примеры команд:**
   - "Привет"
   - "Сколько времени?"
   - "Открой Ютуб"
   - "Громкость вверх"
3. **Пользовательские команды:** Нажмите иконку 📝, чтобы добавить свою команду
4. **Настройки:** Нажмите иконку ⚙️ для изменения устройства ввода, голоса, API-ключей и т.д.

### Сборка EXE
1. Скачайте Silero модель (если нужно офлайн ТТС)
2. Запустите `python build_exe.py`
3. Готовая сборка в `dist/NexusAI/`

## 11. Тестирование

При тестировании проверялись следующие сценарии:
1. Распознавание wake word (проверка на разных расстояниях и фоновом шуме)
2. Выполнение стандартных команд (время, дата, открытие сайтов)
3. Выполнение пользовательских команд
4. Синтез речи (Silero и Edge TTS)
5. Сборка и запуск EXE-файла
6. Работа без интернета (офлайн режим с Silero)
7. Обработка ошибок (нет микрофона, нет интернета, ошибки в конфиге)

Все ключевые сценарии были успешно протестированы. При ошибках пользователю показывается сообщение в чате (красный блок по центру), программа не падает.
